# Инструкция по развертыванию на VPS (Aeza / Timeweb)

Для стабильной работы сайта в РФ и отсутствия блокировок лучше всего использовать российский хостинг с защитой.

## 1. Выбор хостинга (VPS)

Рекомендую **Aeza.net** или **Timeweb Cloud**.
*   **Aeza:** Отличная защита от DDoS (важно для игровых проектов), мощные процессоры, серверы в Москве. Тариф "Promo" или самый дешевый "Shared" (от ~100-200 руб/мес).
*   **Timeweb:** Очень простой интерфейс, надежность.

**Что выбрать при заказе:**
*   **OS (Операционная система):** Ubuntu 22.04 (или 24.04).
*   **Локация:** Москва (для минимального пинга) или Стокгольм/Хельсинки (если нужна Европа).

После оплаты вам на почту придет IP-адрес сервера, логин (обычно `root`) и пароль.

## 2. Подключение к серверу

Вам понадобится программа для подключения.
*   **Windows:** Используйте **PowerShell** (встроен) или программу **MobaXterm** (рекомендую, там есть и консоль, и передача файлов).

Откройте терминал и введите:
```bash
ssh root@ВАШ_IP_АДРЕС
```
(Например: `ssh root@45.12.34.56`)
Введите пароль (он не отображается при вводе) и нажмите Enter.

## 3. Установка Docker (одной командой)

Скопируйте и вставьте эту команду в консоль сервера:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sh get-docker.sh && apt install -y docker-compose-plugin
```

## 4. Загрузка файлов сайта

Самый простой способ — использовать **MobaXterm** или **WinSCP**.
1.  Подключитесь к серверу.
2.  Слева будет список папок сервера. Перейдите в папку `/root` (или просто `/`).
3.  Создайте папку `facevosait`.
4.  Перетащите **все файлы вашего проекта** из папки `facevosait` (где лежат `app.py`, `Dockerfile`, `docker-compose.yml`, `requirements.txt` и т.д.) в эту папку на сервере.

**Важно:** Не забудьте перенести файл `database.db`, чтобы сохранить пользователей и матчи!

## 5. Запуск сайта

В консоли сервера перейдите в папку проекта:
```bash
cd /root/facevosait
```

Запустите сайт командой:
```bash
docker compose up -d --build
```

## 6. Готово!

Сайт будет доступен по адресу: `http://ВАШ_IP_АДРЕС` (без порта, просто IP).

---

### Полезные команды:

*   **Перезапустить сайт (после обновления файлов):**
    ```bash
    docker compose restart
    ```
    или для полной пересборки:
    ```bash
    docker compose up -d --build
    ```

*   **Посмотреть логи (ошибки):**
    ```bash
    docker compose logs -f
    ```
